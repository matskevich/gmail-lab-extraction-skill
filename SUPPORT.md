# support

use GitHub as the source of truth.

## where to ask

- bug: open a bug report issue
- provider/lab support: open a provider request issue
- setup question: open a help request issue or discussion
- idea: start a discussion
- success case: open a success case issue or discussion
- security/privacy issue: follow `SECURITY.md`

## privacy first

do not send raw medical files, screenshots with personal data, portal links, cookies, tokens, or unredacted manifests.

the maintainer cannot safely debug from private medical data in public. the best support artifact is a redacted manifest row plus the command and status.
